# 駅天気アプリ 試作品 企画書

---

## 1. プロジェクト概要

路線図の駅をタップするだけで、その駅周辺の天気・気温・降水確率と服装レコメンドが確認できるWebアプリの試作品。

- **対象路線**：山手線 / 中央線快速（東京〜高尾） / 総武快速線・横須賀線（千葉〜久里浜）
- **対象デバイス**：PCブラウザ（横長レイアウト）
- **公開範囲**：少人数へのURL共有（非公開）
- **目的**：Web技術・外部API連携・セキュリティの学習

---

## 2. 画面レイアウト

```
┌─────────────────────────┬──────────────────────┐
│                         │  【入力エリア】       │
│   【路線図エリア】      │  時間スライダー       │
│   山手線 + 中央線       ├──────────────────────┤
│   + 総武快速・横須賀線  │  【表示エリア】       │
│   （SVG）               │  駅名・天気情報       │
│                         │  服装レコメンド       │
└─────────────────────────┴──────────────────────┘
```

- 左側：路線図（SVG）
- 右上：入力エリア（時間スライダー）
- 右下：表示エリア（天気情報・服装レコメンド）

## 3. 各エリアの仕様

### 3-1. 路線図エリア（SVG）

| 項目 | 内容 |
|------|------|
| 描画方式 | SVGによる模式図（地理的な正確さより見やすさを優先） |
| 対象路線 | 山手線（環状）、中央線快速（東京〜高尾）、総武快速・横須賀線（千葉〜久里浜） |
| 路線の色分け | 山手線：緑、中央線：オレンジ、総武快速・横須賀線：紺/青（JR標準色に準拠） |
| 乗換駅の表示 | 複数路線が交わる駅（東京・新宿・秋葉原など）は1つの点で共有表示 |
| インタラクション | 駅をタップ → 駅名を取得 → 表示エリアに反映 |
| 駅座標データ | 起動時にHeartRails Express APIから取得（静的埋め込み不要） |

**総武快速・横須賀線の扱い**
- 路線図上は「1本の路線」として描画（直通運転のイメージ）
- 千葉〜東京〜久里浜を1本の線でつなぐ
- 東京駅を折り返し点として、路線図上でわかりやすく表現する

**主な乗換駅（複数路線が交わる駅）**

| 駅名 | 乗換路線 |
|------|---------|
| 東京 | 山手線・中央線・総武快速横須賀線 |
| 新宿 | 山手線・中央線 |
| 秋葉原 | 山手線・総武快速横須賀線 |
| 御茶ノ水 | 山手線（接続）・中央線・総武快速横須賀線 |
| 品川 | 山手線・総武快速横須賀線 |

### 3-2. 入力エリア

| 項目 | 内容 |
|------|------|
| コントロール | 時間スライダー（`<input type="range">`） |
| 範囲 | 0時間後〜APIの対応上限（Open-Meteoは168時間≒7日間対応） |
| 表示形式 | 「現在」「3時間後」「24時間後」など、わかりやすいラベルで表示 |

### 3-3. 表示エリア

**初期状態（駅未選択時）**
- 「路線図から駅を選んでください」などのメッセージを表示

**駅選択後の表示内容**

| 表示項目 | 内容 |
|----------|------|
| 駅名 | タップした駅の名前 |
| 日時 | スライダーで選択した日時（例：2026/06/16 15:00） |
| 天気 | 天気の概況（晴れ・曇り・雨 など） |
| 気温 | その時点の気温（℃） |
| 降水確率 | その時点の降水確率（%） |
| 服装レコメンド | 絵文字・アイコンを使った視覚的な表示（例：🧥 コートが必要です） |

## 4. 技術構成

### 4-1. フロントエンド

| 項目 | 技術 |
|------|------|
| 言語 | HTML / CSS / JavaScript（またはReact） |
| 路線図 | SVG手書き（模式図） |
| スタイル | レスポンシブ不要（PC横長レイアウト固定） |

### 4-2. 使用API一覧

| 用途 | API | 料金 | APIキー |
|------|-----|------|---------|
| 駅名→緯度経度 | [HeartRails Express](https://express.heartrails.com/api.html#station) | 完全無料 | 不要 |
| 天気取得 | [Open-Meteo](https://open-meteo.com/) | 完全無料 | 不要 |
| 服装レコメンド | Gemini API（Google） | 無料枠あり | **必要**（要管理） |

#### HeartRails Express API（駅情報取得）

路線名を指定すると、その路線の全駅の名前・緯度経度・前後の駅名を一括取得できる。

```
# 山手線の全駅を取得
GET https://express.heartrails.com/api/json?method=getStations&line=JR山手線

# 中央線快速の全駅を取得
GET https://express.heartrails.com/api/json?method=getStations&line=JR中央線(快速)

# 総武快速線の全駅を取得
GET https://express.heartrails.com/api/json?method=getStations&line=JR総武線快速

# 横須賀線の全駅を取得
GET https://express.heartrails.com/api/json?method=getStations&line=JR横須賀線
```

> ⚠️ **要確認**：HeartRails ExpressのAPIで認識される路線名は、上記と異なる可能性がある。
> コード実装前に必ず下記URLでブラウザから路線名の正式表記を確認すること。
> `https://express.heartrails.com/api/json?method=getLines&prefecture=東京都`
> `https://express.heartrails.com/api/json?method=getLines&prefecture=神奈川県`
> `https://express.heartrails.com/api/json?method=getLines&prefecture=千葉県`

レスポンスの主なフィールド：

| フィールド | 内容 |
|-----------|------|
| `name` | 駅名 |
| `x` | 経度（世界測地系） |
| `y` | 緯度（世界測地系） |
| `prev` / `next` | 前後の駅名（路線順序の把握に利用可能） |
| `line` | 路線名 |

> **利用上の注意**：アプリ内に「HeartRails Express」のクレジット表記が必要。商用・非商用問わず無料で利用可。

#### Open-Meteo API（天気取得）

緯度経度・取得したい気象データの種類・時間単位を指定してリクエストする。

```
# 例：東京駅周辺の気温・降水確率・天気コードを1時間単位で取得
GET https://api.open-meteo.com/v1/forecast
  ?latitude=35.6812
  &longitude=139.7671
  &hourly=temperature_2m,precipitation_probability,weathercode
  &timezone=Asia/Tokyo
```

| 取得データ | フィールド名 | 説明 |
|-----------|-------------|------|
| 気温 | `temperature_2m` | 地上2mの気温（℃） |
| 降水確率 | `precipitation_probability` | 降水確率（%） |
| 天気コード | `weathercode` | 0=快晴、61=小雨、など（WMO規格） |

最大168時間（7日間）先まで取得可能。

### 4-3. データフロー

```
[アプリ起動時]
  ↓
[HeartRails Express API] × 4回
  → 山手線・中央線・総武快速線・横須賀線の全駅情報を一括取得
  → 同名駅を1レコードに統合し、SVG路線図の駅データとして保持

[ユーザー操作]
  ↓ 駅タップ + 時間スライダー選択
[Open-Meteo API]
  → 駅の緯度経度・選択時間で天気・気温・降水確率を取得
  ↓
[Gemini API]
  → 天気情報をプロンプトとして渡し、服装レコメンドを生成（絵文字含む）
  ↓
[表示エリア] → 結果を表示
```

> **起動時に全駅データを取得する設計**のため、駅をタップするたびにHeartRails APIを叩く必要がなく、レスポンスが速い。

## 5. セキュリティ方針

試作品・学習目的のため、最低限のセキュリティ知識を身につけることを目標とする。

### 5-1. APIキー管理の基本

| 方法 | 説明 |
|------|------|
| `.env` ファイル | APIキーを環境変数として別ファイルに記述し、コードに直接書かない |
| `.gitignore` | `.env` ファイルをGit管理対象から除外し、GitHubに漏洩しないようにする |
| 環境変数の読み込み | Node.jsなら `dotenv` パッケージで `.env` を読み込む |

### 5-2. フロントエンドでのAPIキー扱いについて

- フロントエンド（HTML/JS）にAPIキーを直接埋め込むと、ブラウザの開発者ツールから誰でも見られる
- 試作品として割り切る場合はそれでも動作はするが、URLを共有する際はリスクを認識する
- より安全にするには、簡易的なバックエンド（Node.js + Expressなど）を用意し、APIキーはサーバー側のみに置く構成が望ましい

### 5-3. 推奨する学習ステップ

1. まずは `.env` + `.gitignore` の運用を習得する
2. フロントエンド直埋め込みで動くものを作る
3. 余裕があれば簡易バックエンド経由に移行してみる

## 6. 駅データ（設計メモ）

**アプリ起動時にHeartRails Express APIから一括取得**し、メモリ上に保持する。複数路線に属する同名駅は路線をまとめて1レコードに統合する。

```javascript
// 起動時にAPIから取得・整形後のデータ構造（例）
const stations = [
  { name: "東京",  lat: 35.6812, lng: 139.7671, lines: ["JR山手線", "JR中央線(快速)", "JR総武線快速", "JR横須賀線"] },
  { name: "新宿",  lat: 35.6896, lng: 139.7006, lines: ["JR山手線", "JR中央線(快速)"] },
  { name: "品川",  lat: 35.6284, lng: 139.7387, lines: ["JR山手線", "JR横須賀線"] },
  { name: "秋葉原", lat: 35.6982, lng: 139.7732, lines: ["JR山手線", "JR総武線快速"] },
  { name: "高尾",  lat: 35.6426, lng: 139.2828, lines: ["JR中央線(快速)"] },
  { name: "千葉",  lat: 35.6137, lng: 140.1114, lines: ["JR総武線快速"] },
  { name: "久里浜", lat: 35.2196, lng: 139.7187, lines: ["JR横須賀線"] },
  // ...
];
```

**APIリクエストは4回のみ（起動時）：**
1. `getStations&line=JR山手線` → 山手線の全駅を取得
2. `getStations&line=JR中央線(快速)` → 中央線快速の全駅を取得
3. `getStations&line=JR総武線快速` → 総武快速線の全駅を取得（路線名要確認）
4. `getStations&line=JR横須賀線` → 横須賀線の全駅を取得（路線名要確認）

取得後、`prev` / `next` フィールドを使って路線の駅順を把握し、SVGの線を描画する順序として利用できる。

## 7. 未決事項・今後の検討

| 項目 | 内容 |
|------|------|
| HeartRails APIの正式路線名 | 総武快速線・横須賀線の路線名文字列を事前にブラウザで確認する必要あり |
| 時間スライダーの刻み | 1時間単位か3時間単位か（Open-Meteoは1時間単位で取得可能） |
| 服装レコメンドのプロンプト設計 | Geminiへの天気情報の渡し方・出力フォーマットの設計 |
| エラーハンドリング | API失敗時・通信エラー時の表示方法 |
| バックエンドの有無 | Gemini APIキーをどこに置くか（フロント直埋め vs 簡易サーバー） |
| SVG路線図のレイアウト | 3路線になったことで地図上の配置が複雑になる。特に総武快速・横須賀線が東京駅で折り返す表現をどうするか |

## 8. スコープ整理

| 対応する | 対応しない |
|----------|------------|
| 山手線・中央線快速（東京〜高尾） | 他路線、各駅停車、特急区間 |
| 総武快速・横須賀線（千葉〜久里浜） | 支線（逗子〜葉山など） |
| PCブラウザ向け横長レイアウト | スマートフォン対応（レスポンシブ） |
| 現在〜最大7日先の天気 | 過去の天気 |
| 1駅ずつの天気表示 | 複数駅の比較表示 |
| 静的な駅データ | リアルタイムの運行情報 |
